Usar Git para llevar un registro de cambios es básicamente obligatorio si quieres trabajar en proyectos de software de forma seria y organizada. Imagínate trabajar horas en tu código, que algo se rompa y no tener forma de volver atrás, con Git eso no pasa porque puedes regresar a cualquier versión anterior como si nada. También es clave cuando trabajas en equipo, porque evita ese caos de no saber quién cambió qué o de que alguien sin querer borre el trabajo de otro. Cada commit es como una foto guardada del proyecto con una nota que explica qué se hizo, lo que hace todo mucho más fácil de entender y seguir. En resumen, Git no es solo una herramienta técnica, es la diferencia entre trabajar de forma profesional o andar perdido entre archivos sin control.
